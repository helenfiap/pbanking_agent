**Azure AI Foundry integration** uses `ChatOpenAI` with `base_url` — not `AzureChatOpenAI`. All Azure models share one endpoint and one API key:

```python
ChatOpenAI(
    model=model,
    base_url=os.getenv("AZURE_OPENAI_ENDPOINT"),  # /openai/v1
    api_key=os.getenv("AZURE_OPENAI_API_KEY"),
    temperature=0,
)
```

This is cleaner than `AzureChatOpenAI` because Foundry exposes an OpenAI-compatible endpoint for all hosted models — including third-party ones like Kimi and Grok.

**Provider resolution order** (thread-safe):
```python
current_provider = (

)
```

### 3. Thread-Safe Gemini Key Rotation

**The problem:** two Gemini API keys = 40 req/day instead of 20. But the naive `itertools.cycle` approach had two bugs (see Bug Graveyard below).

**The fix — lazy init + lock:**
```python
_key_lock  = threading.Lock()
_key_cycle = None  # initialized on first call, not at import time

def _get_gemini_key() -> str:
    global _key_cycle
    with _key_lock:
        if _key_cycle is None:
            keys = [k for k in [
                os.getenv("GOOGLE_API_KEY"),
                os.getenv("GOOGLE_API_KEY_2"),
            ] if k]
            if not keys:
                raise ValueError("Nenhuma GOOGLE_API_KEY configurada no .env")
            _key_cycle = itertools.cycle(keys)
        return next(_key_cycle)
```

Lazy because `load_dotenv()` must run before `os.getenv()` is called. Thread-safe because multiple benchmark threads would otherwise corrupt the iterator state.

### 4. `extract_text()` Helper (`utils/llm_output.py`)

LangChain changed `response.content` from `str` to `list[dict]` in newer versions (content blocks format). Every node that called `.strip()` directly on the response crashed with `'list' object has no attribute 'strip'`.

Created a single utility applied to all 5 nodes:

```python
def extract_text(response) -> str:
    content = response.content
    if isinstance(content, str):
        return content
    if isinstance(content, list):
        for block in content:
            if isinstance(block, dict) and block.get("type") == "text":
                return block.get("text", "")
            if isinstance(block, str):
                return block
    return str(content)
```

Applied in: `planner.py`, `sql_generator.py`, `error_recovery.py`, `response_formatter.py`, `visualization_agent.py`.

### 7. LangFuse Wired

LangFuse callbacks are now wired into `graph.invoke()` via an optional handler:

```python
def get_langfuse_handler(session_id: str = None):
    if LangfuseCallback is None:
        return None
    try:
        return LangfuseCallback(
            secret_key=os.getenv("LANGFUSE_SECRET_KEY"),
            ...
            session_id=session_id,
        )
    except Exception:
        return None  # never crash the agent

config = {"callbacks": [handler]} if handler else {}
agent_graph.invoke(initial_state, config=config)
```

Session IDs:
- Tab 1: `{banker_id}:{provider}` — traces tagged by banker + model
- Tab 2: `benchmark:{provider}` — one trace per model per run

The import path changed between LangFuse v2 and v3. Handled with a try/except chain:
```python
try:
    from langfuse.callback import CallbackHandler as LangfuseCallback
except ImportError:
    try:
        from langfuse.langchain import CallbackHandler as LangfuseCallback
    except ImportError:
        LangfuseCallback = None
```

## Bug Graveyard

Every bug, root cause, and fix — in the order they were hit.

---

### 🪲 `TypeError: 'Timestamp' object is not subscriptable`
**Where:** sidebar history panel  
**Root cause:** `pd.read_json()` auto-parses ISO strings to `pd.Timestamp` objects. `q['timestamp'][:16]` tries to slice a Timestamp like a string.  
**Fix:** `str(q['timestamp'])[:16]` — stringify first.

---

### 🪲 `with_fallbacks(exception_key=...)` KeyError
**Where:** `llm.py` Gemini cascade  
**Root cause:** LangChain's `with_fallbacks(exception_key="error")` requires a dict input state, but our nodes pass strings.  
**Fix:** Removed `exception_key` entirely. `with_fallbacks([fallback])` without it works fine for standard exception-based fallback.

---

### 🪲 Duplicate `LLM_PROVIDER` in `.env`
**Where:** `.env` lines 3 and 9  
**Root cause:** Both set `LLM_PROVIDER`, last one wins — azure always loaded regardless of UI selection.  
**Fix:** Removed the duplicate. Single declaration at top.

---

### 🪲 `ModuleNotFoundError: No module named 'langfuse.callback'`
**Where:** `app.py` import  
**Root cause:** LangFuse v3 moved the class from `langfuse.callback` to `langfuse.langchain`.  
**Fix:** try/except import chain (see LangFuse section above).

---

### 🪲 `gemini-2.0-flash` quota errors (SHUT DOWN)
**Where:** `.env` `GEMINI_MODEL=gemini-2.0-flash`  
**Root cause:** `gemini-2.0-flash` was deprecated and shut down by Google. The `.env` override was keeping it alive even after we updated the code constants.  
**Fix:** Updated `.env` to `GEMINI_MODEL=gemini-2.5-flash`.

---

### 🪲 `'list' object has no attribute 'strip'` — all 7 models failed
**Where:** Every LangGraph node  
**Root cause:** LangChain started returning `response.content` as `list[dict]` (content blocks) instead of `str` in newer model versions. Direct `.strip()` calls on the list crashed.  
**Fix:** `utils/llm_output.py` with `extract_text()` helper applied to all 5 nodes.

---

### 🪲 `generator raised StopIteration` — all 7 models failed in benchmark
**Where:** `ThreadPoolExecutor` benchmark run  
**Two root causes, both required to reproduce:**

**Cause A — `load_dotenv()` called after imports:**  
```python
from agent.graph import agent_graph  # line 11 — imports llm.py
load_dotenv()                         # line 39 — too late!
```
When `llm.py` was imported, `os.getenv("GOOGLE_API_KEY")` returned `None`, so `_gemini_keys = []` and `itertools.cycle([])` was initialized with an empty list.

**Cause B — `os.environ` race condition:**  
7 threads simultaneously ran `os.environ["LLM_PROVIDER"] = provider`, overwriting each other. Azure threads ended up calling `_make_gemini()` with the empty key cycle.

When `next(empty_cycle)` was called inside a LangGraph node (which runs in a generator context), Python's PEP 479 converted the `StopIteration` to `RuntimeError: generator raised StopIteration`.

**Fixes:**
1. Moved `load_dotenv()` to the very top of `app.py`, before all imports
2. Made `_key_cycle` lazy (initialized on first call, not at import time)
3. Replaced `os.environ["LLM_PROVIDER"] = provider` with `_thread_local.provider = provider` in `run_agent_for_provider`

---

### 🪲 `TimeoutError: 1 (of 7) futures unfinished` — lost all results
**Where:** `as_completed(futures, timeout=MODEL_TIMEOUT_S + 5)` in benchmark  
**Root cause:** When one model (Gemini 2.5 Pro, a thinking model with ~120s latency) exceeded the wall-clock timeout, `as_completed` raised `TimeoutError` and crashed the entire block — losing all 6 results already collected.  
**Fix:** Wrapped the `as_completed` loop in `try/except TimeoutError`. On timeout, mark remaining futures as `"timeout após {MODEL_TIMEOUT_S}s"` and continue to display/save whatever completed.

```python
try:
    for future in as_completed(futures, timeout=MODEL_TIMEOUT_S + 5):
        ...
except TimeoutError:
    for future, prov in futures.items():
        if prov not in results:
            results[prov] = {"success": False, "error": f"timeout após {MODEL_TIMEOUT_S}s", ...}
```

Also removed `gemini-pro` from default benchmark selection — thinking models don't belong in a latency comparison.

---

### 🪲 `.env` line 19 trailing `+` breaking dotenv parse
**Where:** `LANGFUSE_BASE_URL` line  
**Root cause:** Copy-paste artifact left a ` +` at end of line. `python-dotenv` couldn't parse it.  
**Error message:** `python-dotenv could not parse statement starting at line 19`  
**Fix:** `sed -i` to strip trailing characters.

---

### 🪲 `gemini-3.1-flash-lite` → 401 UNAUTHENTICATED
**Where:** `gemini-lite` provider in benchmark  
**Root cause:** Model name was invented — Google's actual lite model is `gemini-2.0-flash-lite`. When a non-existent model name is passed, the Generative Language API returns 401 instead of 404.  
**Fix:** `GEMINI_FALLBACK = "gemini-2.0-flash-lite"` in `llm.py`. Also updated the UI label from "Gemini 3.1 Flash Lite" to "Gemini 2.0 Flash Lite".

---

### 🪲 `est_cost` referenced before assignment in summary table
**Where:** Tab 2 summary table loop  
**Root cause:** `est_cost` was only assigned inside the `if r["success"]` block, then used outside it.  
**Fix:** `est_cost = 0.0` default before the conditional.
